# Como Contribuir

## Fluxo Esperado

1. alinhar a mudanca com backlog ou decisao registrada
2. criar uma branch propria para a `task` ou `subtask`
3. atualizar documentacao relevante
4. implementar a alteracao em commits pequenos
5. rodar `cd SIBiSC && npm run qa:ci`
6. validar impacto com TL e QA quando necessario
7. abrir PR com contexto e links

## O Que Toda Contribuicao Deve Informar

- card principal
- prioridade (`P0`, `P1`, `P2` ou `P3`)
- problema que esta resolvendo
- arquivo ou area impactada
- documento atualizado
- risco ou dependencia remanescente

## Regra

Contribuicao tecnica sem documentacao correspondente nao e considerada completa.
