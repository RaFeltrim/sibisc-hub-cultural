O que é uma Linguagem de Marcação?

Uma linguagem de marcação e um sistema de anotações usado para definir a estrutura e o significado do conteúdo em um documento. Diferentemente de linguagens de programação, ela não éxecuta lógica nem toma decisões. Sua função principal e organizar informações, indicando ao computador (ou ao navegador) qual o papel de cada trecho do conteúdo: o que é titulo, o que é paragrafo, o que é imagem, o que é link.

A ideia de marcar textos vem de antes da web. Na decada de 1960, a IBM desenvolveu a GML (Generalized Markup Language) para organizar documentos corporativos. Essa ideia evoluiu para a SGML (Standard Generalized Markup Language), padronizada pela ISO em 1986, que serviu como base para o HTML.

Principais linguagens de marcação usadas na web

  • HTML (HyperText Markup Language) - a linguagem de marcação padrão da web, usada para estruturar páginas e conteúdo.
  • XML (eXtensible Markup Language) - usada para transportar e armazenar dados de forma estruturada entre sistemas.
  • SVG (Scalable Vector Graphics) - linguagem de marcação baseada em XML para descrever graficos vetoriais bidimensionais.
  • Markdown - linguagem de marcação leve, muito usada em documentações e arquivos README de projetos.
  • XHTML - variação mais rigorosa do HTML, que segue regras do XML.

A Diferença entre HTML e Linguagem de Programação

Uma dúvida muito comum entre iniciantes e achar que HTML e uma linguagem de programação. Não é. É importante entender essa distincao porque ela afeta como voce estuda e como entende o desenvolvimento web.

O que uma linguagem de programação faz

Linguagens de programação, como JavaScript, Python e Java, permitem ao desenvolvedor criar lógica computacional. Com elas e possível:

  • Tomar decisões (se isso acontecer, faça aquilo)
  • Repetir ações (faça isso 10 vezes)
  • Armazenar e manipular dados (listas, objetos, variaveis)
  • Responder a eventos (quando o usuário clicar, execute tal função)

O que o HTML faz

O HTML apenas define a estrutura do conteúdo. Ele diz ao navegador: "este texto e um titulo", "isto e um paragrafo", "aqui tem uma imagem", "este trecho e um link". Ele não calcula, não decide e não repete nada sozinho.

Uma analogia simples: se voce pensa em uma casa, o HTML seria a planta baixa, mostrando onde ficam os cômodos, paredes e portas. A linguagem de programação seria o sistema elétrico e hidráulico, que faz as coisas funcionarem de verdade.

A Origem do HTML

O HTML foi criado em 1991 por Tim Berners-Lee, um físico britanico que trabalhava no CERN (Organização Europeia para Pesquisa Nuclear), na Suíça.

Na epoca, Berners-Lee enfrentava um problema pratico: os pesquisadores do CERN tinham dificuldade em compartilhar documentos e dados entre diferentes computadores e sistemas. Para resolver isso, ele propos um sistema baseado em três pilares fundamentais:

  1. HTML - a linguagem para escrever os documentos
  2. HTTP - o protocolo para transmitir esses documentos entre maquinas
  3. URL - o sistema de enderecos para localizar cada documento

Juntos, esses três elementos formaram a base da World Wide Web (WWW). A primeira página web do mundo foi publicada em agosto de 1991, hospedada no computador NeXT de Berners-Lee no CERN. Ela explicava o próprio projeto da web e como participar dele.

A primeira versão do HTML era extremamente simples: possuia apenas 18 tags, incluindo títulos, parágrafos, listas e, mais importante, a tag de link (anchor), que permitia conectar documentos entre si, criando a ideia de hipertexto que da nome a linguagem.

A Evolução do HTML

Desde sua criação em 1991, o HTML passou por diversas versões, cada uma adicionando recursos para acompanhar a evolução da web:

  • HTML 1.0 (1991) - a versão original criada por Tim Berners-Lee. Tinha apenas 18 tags e permitia criar documentos simples com links.
  • HTML 2.0 (1995) - primeiro padrão oficial, publicado pela IETF. Adicionou suporte a formulários, permitindo que usuarios enviassem dados.
  • HTML 3.2 (1997) - padronizado pelo W3C (World Wide Web Consortium). Introduziu tabelas, applets e melhor controle de alinhamento de texto.
  • HTML 4.01 (1999) - grande marco de maturidade. Separou mais claramente estrutura (HTML) de apresentacao (CSS), introduziu suporte a scripts e acessibilidade.
  • XHTML 1.0 (2000) - reformulacao do HTML 4 usando regras mais rigorosas do XML. Exigia que todas as tags fossem fechadas e escritas em minusculas.
  • HTML5 (2014) - versão atual e revolucionaria. Introduziu tags semânticas (header, footer, article, section, nav), áudio e vídeo nativos, canvas para graficos, APIs para geolocalizacao, armazenamento local e muito mais.

Hoje, o HTML5 e mantido como um "padrão vivo" (Living Standard) pelo WHATWG (Web Hypertext Application Technology Working Group), o que significa que ele e atualizado continuamente, sem versões numeradas futuras.

A Contribuicao do HTML para a Web Atual

O HTML é a espinha dorsal da web. Sem ele, não éxistiria uma forma padronizada de criar e exibir conteúdo nos navegadores. Suas contribuicoes fundamentais incluem:

  • Acessibilidade universal: por ser um padrão aberto e gratuito, qualquer pessoa pode criar uma página web sem licencas ou custos.
  • Semantica: com o HTML5, os desenvolvedores podem dar significado ao conteúdo (nav para navegação, article para artigos, aside para conteúdo lateral), o que melhora tanto a acessibilidade para leitores de tela quanto o SEO (otimização para buscadores).
  • Multimedia nativa: tags como áudio e vídeo eliminaram a necessidade de plugins como Flash, tornando o conteúdo mais seguro e leve.
  • Formulários avançados: campos como email, date, range e number facilitam a entrada de dados e a validação sem precisar de JavaScript.
  • Base para frameworks modernos: tecnologias como React, Angular e Vue geram HTML dinamicamente. Mesmo os aplicativos web mais complexos do mundo, no final das contas, produzem HTML para o navegador renderizar.
  • Interoperabilidade: uma página HTML funciona em qualquer navegador, qualquer sistema operacional e qualquer dispositivo, do smartphone ao computador.

Como um Site e Carregado: O Fluxo Completo

Quando voce digita um endereco na barra do navegador e pressiona Enter, uma serie de etapas acontece nos bastidores. Vamos entender cada conceito envolvido:

Os conceitos-chave

Navegador (Browser)

O navegador é o programa que voce usa para acessar a web. Exemplos: Google Chrome, Firefox, Safari, Edge. Ele tem duas funções principais: enviar pedidos (requisições) para servidores e interpretar as respostas (HTML, CSS, JavaScript, imagens) para exibir a página na tela.

Servidor Web

O servidor é um computador que fica ligado 24 horas por dia, conectado a internet, armazenando os arquivos de um site. Quando recebe um pedido de um navegador, ele localiza o arquivo solicitado e o envia de volta. Exemplos de softwares de servidor: Apache, Nginx, IIS.

URL (Uniform Resource Locator)

A URL e o endereco completo de um recurso na web. Ela tem partes bem definidas:

  • Protocolo: https:// (como a comunicação vai acontecer)
  • Dominio: www.exemplo.com (nome do servidor)
  • Caminho: /página/sobre (qual recurso especifico)

Exemplo completo: https://www.exemplo.com/noticias/artigo-1

HTTP (HyperText Transfer Protocol)

HTTP é o protocolo (conjunto de regras) que define como navegador e servidor se comunicam. O HTTPS e a versão segura, com criptografia, usada pela maioria dos sites hoje. Os métodos HTTP mais comuns são:

  • GET - pede um recurso (quando voce acessa uma página)
  • POST - envia dados (quando voce preenche e envia um formulario)
  • PUT - atualiza um recurso existente
  • DELETE - remove um recurso

O fluxo de Requisicao e Resposta passo a passo

  1. Voce digita a URL no navegador, por exemplo: https://www.sibisc.com.br/noticias
  2. Resolucao DNS: o navegador consulta o DNS (Domain Name System), que funciona como uma lista telefonica da internet, para descobrir o endereco IP numerico do servidor (por exemplo, 203.0.113.42).
  3. Conexao TCP: o navegador estabelece uma conexao com o servidor usando o protocolo TCP (Transmission Control Protocol), que garante entrega confiavel dos dados.
  4. Requisicao HTTP (Request): o navegador envia uma mensagem ao servidor dizendo: "Quero o recurso /noticias, usando o metodo GET, via HTTP/2".
  5. Processamento no Servidor: o servidor recebe a requisicao, localiza o recurso pedido (pode ser um arquivo HTML estatico ou uma página gerada dinamicamente por codigo do servidor), e prepara a resposta.
  6. Resposta HTTP (Response): o servidor envia de volta o arquivo HTML junto com um codigo de status. Os mais comuns são:
    • 200 OK - tudo certo, recurso encontrado
    • 301 Moved Permanently - o recurso mudou de endereco
    • 404 Not Found - recurso não éncontrado
    • 500 Internal Server Error - erro no servidor
  7. Renderizacao: o navegador recebe o HTML e comeca a interpreta-lo. Ao encontrar referências a outros recursos (CSS, JavaScript, imagens, fontes), faz novas requisições para cada um deles.
  8. Página pronta: quando todos os recursos são carregados e processados, o navegador exibe a página final na sua tela.
