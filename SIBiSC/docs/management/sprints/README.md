# Sprints do MVP

## Objetivo

Organizar o MVP em sprints pequenas, progressivas e didaticas. Esta trilha foi pensada para alunos com pouca experiencia, priorizando aprendizado continuo e entregas que se encaixem sem conflitos.

## Sequencia Recomendada

1. [`sprint_00_setup_e_onboarding.md`](./sprint_00_setup_e_onboarding.md)
2. [`sprint_01_fundacao_web_mobile.md`](./sprint_01_fundacao_web_mobile.md)
3. [`sprint_02_noticias_e_eventos.md`](./sprint_02_noticias_e_eventos.md)
4. [`sprint_03_acervo_e_geolocalizacao.md`](./sprint_03_acervo_e_geolocalizacao.md)
5. [`sprint_04_integracao_qa_e_release_mvp.md`](./sprint_04_integracao_qa_e_release_mvp.md)

## Como Usar

- o PM usa estas sprints para organizar o cronograma
- o TL usa estas sprints para controlar dependencia e risco tecnico
- os alunos usam estas sprints para entender o que aprender e o que entregar em cada fase

## Regra Importante

Nao iniciar uma sprint nova sem confirmar:

- dependencias da sprint anterior resolvidas
- documentacao minima atualizada
- entregas testadas no fluxo principal
- cards grandes quebrados em tasks e subtasks menores
