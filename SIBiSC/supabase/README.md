# Supabase Workspace

Esta pasta será usada para:

- config do Supabase
- migrations
- seeds
- edge functions
- documentação operacional associada

## Objetivo

Servir como espaço versionado para a base de dados e integrações server-side do SIBiSC.

## Regras

- alterações de banco devem ser versionadas
- RLS deve nascer junto com o schema
- nenhuma mudança crítica deve existir apenas no painel visual
- o conteúdo daqui deve estar alinhado com `SIBiSC/docs/data` e `SIBiSC/docs/backend`
