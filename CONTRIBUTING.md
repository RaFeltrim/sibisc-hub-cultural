# Contributing

## Objetivo

Este repositório existe para organizar e evoluir o projeto **SIBiSC** com documentação clara, decisões rastreáveis e entregas incrementais.

## Regras Gerais

1. Toda decisão relevante deve ser registrada em `SIBiSC/docs`.
2. Toda documentação oficial deve nascer em `.md`.
3. Referências externas e acadêmicas não substituem a documentação do projeto.
4. Mudanças em produto, arquitetura, banco, QA ou deploy devem atualizar o documento correspondente.

## Fluxo Recomendo

1. alinhar a mudança com uma US, épico ou decisão registrada
2. atualizar ou criar a documentação necessária
3. implementar a alteração técnica
4. revisar impactos cruzados com a `Tech Lead`
5. abrir PR com contexto claro

## Convenções

- use nomes de arquivos descritivos e em minúsculas com `_`
- mantenha a documentação específica ao SIBiSC
- trate `_references/` como apoio, não como fonte oficial do produto

## Pull Requests

Todo PR deve informar:

- o que mudou
- por que mudou
- quais documentos foram atualizados
- quais riscos ou dependências permanecem abertos
